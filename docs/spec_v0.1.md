# 1ページ仕様 v0.1
## 中隊規模・現代陸戦・司令部視点RTS / コンクエスト

---

## コンセプト

不完全な情報（観測・誤認・遅延）の中で、指揮・偵察・火力・補給の摩擦を管理し、複数拠点の保持で勝利する「現代戦術級（中隊規模）」リアルタイム指揮ゲーム。

> ※基地建設なし（資源採取・施設建設・生産ツリーなし）。増援は上級司令部からの逐次投入として扱う。

---

## マッチ

| 項目 | 値 |
|------|-----|
| 形式 | 1v1（将来2v2拡張可） |
| 試合時間 | 最大30分（平均決着20〜30分想定） |
| 初期チケット | 600 vs 600 |

**勝利条件（どちらか先に達成）**
1. 敵チケットを0にする
2. 時間終了時に自軍チケットが多い

---

## マップ / 拠点（コンクエスト）

### 拠点構成
- **拠点数**：5（A〜E）
- **初期支配**：両軍それぞれ端の1拠点を保持、残り3拠点は中立

### 機能ノード（各拠点に最低1つ配置）

| ノード | 効果 |
|--------|------|
| COM（通信） | 情報共有・命令遅延に影響 |
| LOG（交通/兵站） | 増援到着遅延に影響（受け入れ口） |
| OBS（観測） | 索敵/同定の有利（更新頻度・距離補正） |

### 制圧ルール（歩兵優位を固定）

| 状態 | 制圧速度 |
|------|---------|
| 標準（歩兵1ユニット） | 通常 |
| 抑圧（Suppressed）中 | -50% |
| 退避/伏せ（Pinned相当） | -80% |

- **中立化**：25秒 ＋ **奪取**：35秒 ＝ 合計 60秒（基準）
- **制圧可能**：歩兵系（歩兵/工兵/偵察）
- **車両**：制圧不可（ただし"妨害＝膠着"は可能）
- 敵味方同時侵入でゲージ膠着

---

## スコアリング（チケット出血）

**10秒ごとに、拠点数が少ない側が出血：**

```
減少 = 2 × (拠点差)
```

**例**：2拠点 vs 3拠点 → 負け側は10秒で-2（1分で-12）

> 追加の戦闘損耗はチケットに直結させない（殲滅ゲー化防止）。勝敗は「保持」に寄せる。

## 増援経済（RP：Requisition Points）

| 項目 | 値 |
|------|-----|
| 開始RP | 1000 |
| RP上限（貯金上限） | 2000 |
| 収入 | 10秒ごとに `10 + 5 × (保持拠点数)` |
| 拠点奪取ボーナス | +80RP（奪取時1回） |

> 意図：拠点保持が増援テンポを生むが、貯め込み一気出しを抑制する。

## 召喚（増援投入）＝「到着計画」

増援は購入即出現ではなく **「到着予約」方式**。

### 到着地点
- 自軍保持かつ非コンテストの拠点を指定
- LOG拠点は特に有利

### 到着遅延（カテゴリ基準）

| カテゴリ | 遅延 | 輸送枠 |
|---------|------|--------|
| 軽 | 20秒 | 1枠 |
| 中 | 35秒 | 1枠 |
| 重 | 50秒 | 2枠 |

- **輸送枠**：同時投入制限 = **2スロット**
- **迂回ペナルティ**：到着予定拠点がコンテスト化した場合、最寄り安全拠点へ自動迂回し +15秒

## 情報（Fog of War + 不確実性）

### 敵情報の確度（3段階）

| 状態 | 表示 | 意味 |
|------|------|------|
| CONF（確定） | 通常記号 | 視認/確実センサーで更新中 |
| SUS（推定） | 薄い記号＋不確定楕円 | 最終確認地点＋不確定範囲 |
| LOST（消失） | 非表示 | 一定時間更新なし |

### 陳腐化タイミング（基準）
- 最終更新から **15秒** → CONF → SUS
- 最終更新から **60秒** → SUS → LOST

### 情報フロー
```
観測（ユニット視界/センサー）
  → 報告
  → 共有（COM拠点/HQリンクの通信状態で速度変化）
```

## 指揮摩擦（C2）

### 命令遅延（基準）

| 通信状態 | 遅延 |
|---------|------|
| 良好 | 2秒 |
| 不安定 | 6秒 |
| 途絶 | 命令不可（ローカル命令のみ） |

### 命令種別
`Move / Attack / Defend / Recon / Resupply / Dig-in` など

### 途絶時のSOP（標準行動）

| 状況 | 自律行動 |
|------|---------|
| 交戦中 | 遮蔽へ移動 → 抑圧低下を待つ → 最後の命令地点に留まる |
| 非交戦 | その場防御（Defend）に移行 |

> ※SOPはv0.1固定。将来カスタム可能。

## 戦闘の骨格

### ユニット状態（2本軸）

| 状態値 | 範囲 | 意味 |
|--------|------|------|
| **Strength**（稼働率/人員） | 0〜100 | 戦闘力・機動力の基礎 |
| **Suppression**（抑圧） | 0〜100 | 射撃・移動・制圧への制約 |

### 抑圧閾値

| 閾値 | 状態 | 効果 |
|------|------|------|
| 40 | Suppressed | 命中・移動低下 |
| 70 | Pinned | 移動ほぼ不可、制圧ほぼ停止 |
| 90 | Broken | 後退行動、回復まで戦闘力極小 |

### カバー（遮蔽）
地形により被害と抑圧が大きく変化する（地形が戦力）。

### 回復ルール

| 回復対象 | 回復条件 |
|---------|---------|
| Suppression | 非被弾・通信良好・HQ近傍で早く回復 |
| Strength | 「再編/補給」アクションが必要（**自然回復なし**） |

---

## 持続（補給・再編）

弾薬/燃料は詳細数値ではなく **「持続力ゲージ（Sustain）」** として抽象化（v0.1）。

**Sustain低下時の影響：**
- 発射頻度の低下
- 機動力の低下
- 回復速度の悪化

**Resupply方法：**
- 補給ユニット（後で定義）
- LOG拠点近傍での Resupply アクション

> 意図：前線を出しっぱなしにすると鈍り、交代・補給の判断が必須になる。

## UI（兵科記号前提）

### 兵科記号ライブラリ

**[milsymbol](https://github.com/spatialillusions/milsymbol)**（MIT License）を採用。

- NATO APP-6 / MIL-STD-2525D 準拠のSVG記号を生成するJavaScriptライブラリ
- `tools/generate_symbols.js` で必要な兵科記号SVGを事前生成し `assets/units/symbols/` に配置
- Godotはその静的SVGをテクスチャとして読み込む（ランタイムJS実行なし）
- 所属（味方/敵/不明）×確度（CONF/SUS）のバリアントを生成

**生成手順:**

```bash
cd tools
npm install
node generate_symbols.js
```

兵科記号＋重ね表示のみで成立させる。

### 必須表示
- Strengthバー
- Suppressionバー
- 情報確度（CONF/SUS/LOSTの見え方差）
- 通信状態（良好/不安定/途絶）
- 命令状態（実行中/遅延/自律SOP）
